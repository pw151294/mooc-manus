## 整体任务目标
基于 mooc-manus 现有智能体对话主链路 `/api/agent/chat`，**无大规模重构核心对话框架**，落地面向 Bash 工具调用的 **Human-in-the-Loop 人工介入审批机制**，解决编程智能体场景下 Shell 命令高危执行风险；同时配套前端SSE事件推送、新增续行接口、会话阻塞与恢复逻辑，并输出完整开发任务清单 + 接口/数据结构规范 + 全链路测试方案与自动化E2E验证。

## 一、业务背景与核心诉求
mooc-manus 定位逐步迭代为代码编程类智能体，会高频调用 Bash 工具执行服务器Shell指令；硬编码黑名单/正则匹配高危命令存在覆盖不全、无法适配动态拼接脚本、上下文意图无法识别等安全漏洞。
因此不采用静态规则拦截，改用**大模型自主识别脚本风险 + 人工审批阻断**的人机闭环方案：
1. 由智能体自身推理判定待执行Bash脚本是否具备操作风险；
2. 高危调用强制打断工具执行流程，推送审批事件至前端；
3. 用户手动选择「执行」/「拒绝」后，通过独立接口续行对话会话，继续或终止本次工具调用，恢复原有SSE流式对话链路。

## 二、标准化详细实现方案与约束
### 1. 新增核心事件定义：工具调用中断审批事件
新增SSE推送事件枚举类型：`TOOL_CALL_INTERRUPT`
事件固定数据结构必须包含字段：
- traceId：本次对话会话唯一标识
- callId：单条工具调用唯一ID
- toolName：工具标识（固定为bash执行工具）
- params：原始调用入参（完整shell脚本内容）
- riskAnalysis：大模型输出的风险判定理由、具体风险点说明

### 2. 前置风险判定与流程阻断逻辑（核心）
1. 智能体LLM输出Action明确要调用Bash工具后，**不直接下发远程SSH执行指令**；
2. 插入一轮内嵌推理：让大模型基于待执行Shell脚本自主分析是否存在删除文件、强制修改系统配置、高危提权、批量删数据、挂载磁盘、覆盖系统文件、外联恶意地址等高风险行为；
3. 若判定为**存在风险**：
    - 立即终止本轮工具调用下发流程；
    - 构造 `TOOL_CALL_INTERRUPT` 事件通过SSE推送到前端；
    - 阻塞当前对话SSE工作流，暂停后续大模型推理与工具调度，会话挂起等待外部人工指令；
4. 若判定无风险：直接走原有逻辑执行工具调用，无需人工介入。
> 硬性约束：全程不使用硬编码黑名单、正则规则、配置表做命令拦截，风险识别能力完全交由LLM自主判断；原有Agent主调度框架、上下文结构、SSE基础流式输出逻辑尽量最小侵入改造。

### 3. 前端交互规范
前端监听SSE流捕获 `TOOL_CALL_INTERRUPT` 事件后，在对话消息区渲染审批卡片：
- 展示工具名称、完整Shell脚本内容、AI给出的风险分析提示；
- 提供两个可交互按钮：【确认执行】、【拒绝本次调用】；
- 点击按钮后携带会话标识、callId、用户操作类型调用后端新增续行接口。

### 4. 新增后端接口设计：`POST /api/agent/resume`
#### 接口入参约定
```json
{
  "traceId": "会话全局ID",
  "callId": "被阻塞的单条工具调用ID",
  "action": "execute | reject"
}
```
#### 接口核心职责（本次开发三大技术难点重点落地）
1. 根据 traceId 寻址到当前处于挂起阻塞状态的对话协程/会话上下文；
2. 注入用户人工决策结果，解除当前SSE工作流阻塞；
3. 分支处理：
    - action=execute：放行该条Bash调用，走原有工具执行逻辑，执行完毕后继续后续正常对话流程；
    - action=reject：标记本次工具调用被人工否决，将拒绝结果写入对话上下文，告知大模型本次调用已终止，由Agent重新规划后续任务，SSE流恢复继续输出。

### 5. 会话生命周期约束
- 仅单条Bash工具调用会触发单次阻塞审批；
- 同一会话可多次触发HITL人工审批；
- 会话正常结束、前端手动关闭对话、会话超时自动销毁时，自动清理挂起阻塞态，释放资源。

    - 风险判定Prompt渲染校验、事件结构体序列化校验、resume接口参数校验、会话状态流转单元用例；
2. **集成测试**
    - 模拟Agent输出Bash调用→触发风险识别→推送中断事件→会话阻塞→调用resume接口放行/拒绝→校验后续流程流转正确性；
3. **端到端自动化E2E测试**
   基于Playwright + MCP能力编写自动化脚本，完整复现用户侧操作链路：
    - 发起对话会话；
    - 诱导Agent生成高危Shell命令；
    - 断言前端收到审批SSE事件并渲染按钮；
    - 自动化模拟点击执行/拒绝；
    - 断言会话解除阻塞，后续分支逻辑符合预期；
4. 明确该HITL机制的上线验收Pass/Fail判定标准，包含正向用例、反向边界用例（会话超时、重复调用resume、非法callId等异常场景）。

## 四、额外强制约束
1. 禁止改动 mooc-manus 原有 `/api/agent/chat` 核心消息循环大框架，优先采用钩子、中间件、工具调用前置拦截器方式接入能力；
2. 阻塞逻辑不能造成协程泄漏、内存常驻占用，必须配套超时回收机制；
3. 所有新增事件、接口、字段需兼容现有SSE流式输出格式，不破坏前端已有消息解析逻辑。

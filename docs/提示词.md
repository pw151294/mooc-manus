## 一、需求背景
当前 mooc-manus 智能体工具执行报错处理完全依赖大模型自主推理：工具执行失败后仅将错误信息封装为 ToolMessage 塞入上下文，无标准化纠错引导。
该方式对模型推理要求极高，在报错信息不全、非常规异常场景下，容易出现盲目重试、任务卡死、决策错误等问题。

为降低模型负担、提升工具执行稳定性，**基于现有 Skill 能力轻量化接入错误恢复机制**，仅通过编写标准化 `SKILL.md` 实现，不改造底层代码、不新增工具、不侵入原有链路。

## 二、功能定位与触发规则
### 1. Skill 能力定位
为智能体**文件读写编辑、Bash 脚本执行**四类原生工具提供**通用、轻量、分层的错误自愈策略**，规范报错后的下一步行为，避免模型自由发散决策。

### 2. 触发时机（固定写入 SKILL.md）
仅在**原生文件/脚本工具执行失败、返回错误信息时自动触发**；工具执行成功不生效、不干扰正常流程。

## 三、轻量分层错误恢复策略（大幅精简、无细粒度枚举、不限制模型思路）
摒弃海量具体报错枚举，采用**通用三层分类范式**，给予模型充足自主推理空间，仅约束行为边界，不局限思考：

### 层级1：可自愈类错误（自动修复，无需用户干预）
适用于**环境/前置条件缺失类常规报错**：路径不存在、目录缺失、权限不足、依赖缺失、环境配置异常等。
通用策略：自主调用现有原生工具**补齐缺失条件、修复环境状态**后重试执行，允许模型根据报错信息自主选择合适排查与修复手段，不做固定步骤强制绑定。

### 层级2：信息缺失类错误（交互确认，不盲目重试）
适用于**信息模糊、参数不明确、歧义性报错**：路径不唯一、参数缺失、操作目标无法确定等。
通用策略：终止自动重试，整理问题关键点**向用户精准询问补充信息**，待用户确认后再继续任务。

### 层级3：致命不可恢复错误（直接上抛终止）
适用于**无法自愈、无修复空间的异常**：系统权限封禁、资源耗尽、安全拦截、严重语法错误、磁盘异常等。
通用策略：放弃修复与重试，**终止当前消息工具链路**，整理错误原因友好上抛给用户。

## 四、Skill 核心约束（轻量、宽松、保灵活）
1. 所有修复动作**仅复用项目现有四类原生工具**，不使用外部能力；
2. 本 Skill 仅提供**决策分层规范**，不限制具体报错细节推理逻辑，模型可结合上下文自主分析根因与修复方式；
3. 兼容原有 PlanMode 规划模式：错误处理进度可按需更新 Plan.md / TODO.md；
4. 仅做行为约束，不做细粒度步骤绑定，保留智能体最大思考自由度。

## 五、交付要求
1. 仅产出**一份高质量、轻量化、可直接使用的 SKILL.md**，无需开发任何业务代码；
2. 文档包含：Skill名称、功能简介、触发时机、轻量三层错误恢复范式、模型行为约束；
3. 去除海量报错枚举、去除死板固定步骤，保证上下文极精简、策略通用、适配全部工具异常场景。需要我直接帮你**生成这份精简版最终SKILL.md文件**，可以直接投入项目使用吗？

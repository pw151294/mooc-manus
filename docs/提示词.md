## 正式业务需求提示词
### 一、项目整体说明
基于已有 `mooc-manus` 智能体编排平台后端工程，新建前端项目 `mooc-manus-web`，前后端通过 RESTful API + SSE 事件流完成数据交互，整体定位为**可视化智能体编排管理平台**，实现模型、工具、Skill、智能体对话编排全生命周期可视化配置与调试能力，同时完成多仓库 Git 模块化管理方案落地。

### 二、后端数据映射说明（数据库表关联）
1. 模型配置模块：对接后端 `app_config` 表相关接口，完成模型配置的新增、编辑、删除、列表查询、详情查看；
2. 工具包管理模块：对接 `tool_provider`（工具供应商）、`tool_function`（工具函数/工具包）两张表对应后端接口，实现供应商维护、工具包的增删改查、启用禁用、参数配置管理；
3. Skill 管理模块：对接 `skill`、`skill_version`、`skill_provider` 三张表对应后端接口，实现 Skill 基础信息维护、多版本管理、绑定对应技能供应商、上下线管控；
4. 智能体可视化对话模块：
   1）能力装配可视化面板：支持可视化勾选配置模型参数、已接入工具包、已上架 Skill，配置内容序列化后作为请求参数提交给后端对话接口；
   2）多轮对话聊天窗口：同时兼容**普通非流式返回**、**SSE 流式返回**两种场景；
   3）SSE 场景需前端完成事件流订阅、分片文本实时解析，实现打字机流式逐段渲染智能体回复内容

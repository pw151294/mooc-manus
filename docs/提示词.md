## 一、任务总述
基于现有 `docs/prompts/S2.md` 根因分析系统提示词存量内容，完成**S2智能体提示词架构解耦重构**，核心目标：剥离原系统提示词内多分支耦合的告警分析逻辑，由「单文件全流程硬编码提示词」改造为「轻量化路由型系统提示词 + 四类独立专项Skill」架构，解决长上下文下大模型注意力稀释、多if-else分支逻辑冗余易混淆、流程不可插拔维护的问题。

## 二、现存架构痛点
原 `S2.md` 将日志、链路、指标、主机四类告警根因分析全分支if-else逻辑、执行步骤、分析思维链、输出范式全部内嵌在系统Prompt中。复杂排查会话上下文堆叠后，模型极易丢失核心执行约束，推理准确率下降，且新增/修改单类告警排查流程需整体修改主Prompt，扩展性极差。

## 三、架构改造规范与约束
### 3.1 顶层系统提示词（精简新版 `docs/prompts/S2.md`）
1. 彻底移除原文件内所有分类型告警排查if-else分支流程；
2. 新版S2主Prompt仅保留**身份定位、权限边界、全局通用输出规范、全局异常兜底规则、路由调度逻辑**；
3. 强制固定首步执行链路：
   ```
   收到告警事件 → 调用MCP工具 sres_alert_detail 拉取完整告警元数据 → 识别告警所属分类（日志/链路/指标/主机）→ 精准加载对应专项分析Skill，交由该Skill接管后续全流程根因排查
   ```
4. 主Prompt仅做入口路由与全局管控，不承载任何一类告警的分步排查思维链。

### 3.2 需通过 `skill-creator` 工具批量创建4个标准化Skill
| Skill 标识 | 技能用途 | 数据源来源                             |
| ---- | ---- |-----------------------------------|
| sre_log_analyzer | 日志类告警专属根因分析思维链与执行规范 | 提取原S2.md中日志告警if分支全部分析思维链、自检项、输出格式 |
| sre_trace_analyzer | 调用链链路告警专属根因分析思维链与执行规范 | 提取原S2.md中链路告警if分支全部分析思维链、自检项、输出格式    |
| sre_metric_analyzer | 监控指标类告警专属根因分析思维链与执行规范 | 提取原S2.md中指标告警if分支全部分析思维链、自检项、输出格式    |
| sre_host_analyzer | 服务器主机资源类告警专属根因分析思维链与执行规范 | 提取原S2.md中主机告警if分支全部分析思维链、自检项、输出格式    |

### 3.3 单个Skill文件（SKILL.md）统一编写要求
1. 拆解原S2.md嵌套式if-else条件分支，**扁平化重写为单条线性递进式思维链**，去除多层条件嵌套，降低模型理解成本；
2. 每个Skill仅服务单一告警类型，不混入其他告警场景逻辑，实现上下文最小化，规避注意力稀释；
3. 每个Skill必须完整继承对应分支下：工具调用顺序、数据查询优先级、交叉校验规则、结果自检机制、结构化输出模板、失败降级处理逻辑；
4. Skill内仅包含本场景专属流程，全局通用约束沿用新版S2主系统提示词，无需重复冗余声明。

## 四、分步执行操作指令（CC可直接落地）
1. 读取并解析 `docs/prompts/S2.md` 全文，拆分出**全局公共内容**与**四大告警类型分支私有流程内容**；
2. 基于全局公共内容，重写精简版 `docs/prompts/S2.md`，删除所有告警场景分支逻辑，仅保留路由调度、身份权责、全局输出标准、通用异常处理；
3. 针对四类告警分别剥离对应分支逻辑，重构为线性无嵌套的分步排查流程，生成四份独立SKILL.md文档；
4. 调用内置 `skill-creator` 工具，依次创建 `sre_log_analyzer`、`sre_trace_analyzer`、`sre_metric_analyzer`、`sre_host_analyzer` 四项技能并入库生效；
5. 最终输出交付物清单：
    - 精简后 `docs/prompts/S2.md`
    - 4份独立Skill源码文件，输出到`docs/skills`
    - 技能创建完成回执说明

## 五、硬性约束
- 不得丢失原S2.md任一原有排查规则、校验逻辑、输出格式；
- 禁止将多类告警逻辑合并至同一个Skill；
- 主Prompt只路由不执行业务排查，所有细分排查动作全部下沉至对应Skill；
- 所有嵌套if条件必须拆解为顺序执行判断步骤，禁止多层分支嵌套结构。

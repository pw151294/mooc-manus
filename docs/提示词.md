## 一、项目背景与核心问题
当前MOOC-Manus智能体对话框架已支持**多轮连续对话、实时工具调用、Skill任务执行**核心能力，可满足常规轻量化交互需求。但在**超多轮对话、复杂长周期任务执行**场景下，存在关键技术缺陷：

大模型注意力机制仅聚焦上下文首尾内容，对话轮次堆叠后，**中间核心上下文、任务进度、执行逻辑会被注意力稀释**，导致智能体遗忘前置任务规划、中断执行逻辑、遗漏待办步骤，最终造成复杂任务执行失败、多轮对话任务一致性断裂。

针对该问题，需为框架新增**任务状态管理+记忆持久化**能力，解决长会话、长任务的记忆丢失与上下文稀释问题。

## 二、核心优化方案（工程改造需求）
本次改造核心为**新增规划模式（PlanMode）开关机制**，基于原生文件读写工具实现任务记忆持久化，无需新增工具，完全复用框架原生文件编辑、文件读取、脚本执行Tool能力。

### 1. 新增全局/请求级 PlanMode 开关
在智能体对话请求结构体中，新增 **规划模式标识符（PlanMode）** 字段，字段为布尔值（True/False），用于控制是否开启复杂任务持久化规划能力：
- **关闭状态（默认）**：保留原有框架所有能力，无任何新增逻辑，适配日常简单对话、轻量化任务。
- **开启状态**：激活任务规划与记忆持久化流程，自动注入专属系统提示词，强制智能体遵循标准化任务管理逻辑。

### 2. 规划模式专属系统提示词注入机制
当请求携带 PlanMode=True 时，框架自动追加一段固定系统提示词，赋予智能体**实时任务规划、进度持久化、待办迭代更新**的强制行为意识，提示词核心约束如下：
「当前已开启智能体规划模式，本次任务为长周期/复杂任务，需全程执行记忆持久化操作：1. 任务启动初期，必须基于用户需求梳理完整任务流程，生成标准化 Plan.md 任务规划文档；2. 同步拆解规划内容，生成结构化 TODO.md 待办清单，明确待执行步骤、执行顺序、完成状态；3. 任务每完成一个步骤、迭代一次逻辑、变更一次方案，必须实时更新 Plan.md 与 TODO.md，保证文件内容与实际执行进度完全对齐；4. 会话中断、上下文溢出、重启会话后，需优先读取本地持久化的 Plan.md、TODO.md 文件，恢复任务状态、接续剩余流程，杜绝记忆丢失、任务断层；5. 所有工具调用、脚本执行、文件操作行为，均需围绕两份文档的规划逻辑开展」

### 3. 持久化文件实现规范
- 持久化载体为 **Plan.md（任务规划文件）**、**TODO.md（待办清单文件）**，无自定义文件，完全基于框架**原生文件读取/文件编辑Tool**实现创建、写入、修改、读取操作。
- Plan.md：存储任务整体目标、拆解逻辑、执行流程、技术方案、整体规划大纲，随任务迭代动态更新。
- TODO.md：存储精细化分步待办、步骤执行状态（未执行/执行中/已完成）、步骤执行结果，实时同步任务进度。

## 三、功能验证要求（基于 /goal 技能驱动 Claude Code 完成）
所有验证流程需通过框架原生 **/goal Skill** 驱动执行，落地2个核心验证场景，完整校验「规划模式生效、记忆持久化、任务断点续跑」能力，验证通过需输出完整执行日志与结果证明。

### 验证场景1：常规复杂任务全流程执行验证
#### 触发条件（WHEN）
开启智能体 PlanMode 规划模式，下发任务：创建简易 Golang 工程，编写 main 函数，实现控制台打印「hello world」并成功执行输出。

#### 预期行为（THEN）
1. 智能体接收任务后，优先通过原生文件工具创建并生成完整的 Plan.md 任务规划文档，梳理Golang工程创建、代码编写、脚本执行全流程；
2. 同步创建 TODO.md 待办清单，拆解工程初始化、文件创建、代码编写、运行测试等细分步骤；
3. 严格按照两份文档的规划逻辑，依次调用文件编辑、脚本执行工具完成全流程任务；
4. 最终成功运行代码，控制台正常输出「hello world」，任务完整闭环，无遗漏、无报错。

### 验证场景2：会话中断+项目重启的断点续跑验证（核心记忆持久化校验）
#### 触发条件（WHEN）
开启智能体 PlanMode 规划模式，下发与场景1一致的Golang工程创建任务；待智能体**完成 Plan.md、TODO.md 文件创建与写入，尚未执行代码运行步骤时**，主动终止当前MOOC-Manus对话会话。随后完整重启 MOOC-Manus 后端项目，新建对话会话。

#### 预期行为（THEN）
1. 新项目重启、新会话启动后，无需用户重复描述任务需求；
2. 智能体自动识别规划模式，主动读取本地已持久化的 Plan.md、TODO.md 文件，恢复历史任务状态与未完成进度；
3. 基于读取的任务规划与待办清单，自主接续剩余任务流程，调用对应工具完成代码执行；
4. 最终成功运行Golang代码，控制台正常输出「hello world」，验证**记忆持久化、断点续跑、抗上下文稀释**核心能力生效。

## 四、开发与验证约束规范
1. **无侵入改造**：所有优化逻辑基于原有框架扩展，不修改原有普通对话、工具调用、Skill执行的核心逻辑，保证原有功能兼容可用。
2. **轻量化改造**：仅新增请求开关、系统提示词注入逻辑，完全复用原生文件Tool，不新增自定义工具、不新增冗余模块。
3. **可观测验证**：两个场景验证需全程可追溯，保留文件生成记录、工具调用日志、任务进度更新记录、最终执行输出结果。
4. **闭环落地**：完成代码开发、功能自测、双场景验证，确保彻底解决多轮对话、复杂任务的上下文注意力稀释、任务记忆丢失问题。
